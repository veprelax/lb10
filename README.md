## 2. Теоретичні відомості

### 2.1. Файлова система Linux

Linux використовує ієрархічну файлову систему, де все починається з кореневої директорії `/`. Основні команди для роботи з файловою системою:
- `ls` - перегляд вмісту директорії
- `cd` - зміна поточної директорії
- `mkdir` - створення директорій
- `rm` - видалення файлів
- `chmod` - зміна прав доступу
- `find` - пошук файлів

### 2.2. Управління користувачами

Linux є багатокористувацькою системою. Основні команди:
- `useradd` - створення користувача
- `usermod` - модифікація користувача
- `groupadd` - створення групи
- `passwd` - зміна пароля

### 2.3. Мережеві налаштування

Команди для роботи з мережею:
- `ip` - налаштування мережевих інтерфейсів
- `ping` - перевірка з'єднання
- `ss` / `netstat` - перегляд мережевих з'єднань

### 2.4. Системні сервіси

Сучасні Linux дистрибутиви використовують systemd для управління сервісами:
- `systemctl` - управління сервісами
- `journalctl` - перегляд логів

## 5. Відповіді на контрольні питання

### Питання 1: Які основні команди використовуються для роботи з файловою системою в Linux?

**Відповідь:**

Основні команди для роботи з файловою системою Linux:

1. **Навігація:**
   - `ls` - перегляд вмісту директорії
   - `cd` - зміна поточної директорії
   - `pwd` - показ поточної директорії

2. **Створення та видалення:**
   - `mkdir` - створення директорій
   - `touch` - створення порожніх файлів
   - `rm` - видалення файлів
   - `rmdir` - видалення порожніх директорій
   - `rm -r` - рекурсивне видалення

3. **Копіювання та переміщення:**
   - `cp` - копіювання файлів та директорій
   - `mv` - переміщення/перейменування файлів

4. **Пошук:**
   - `find` - пошук файлів за різними критеріями
   - `locate` - швидкий пошук файлів
   - `which` - пошук виконавчих файлів

5. **Права доступу:**
   - `chmod` - зміна прав доступу
   - `chown` - зміна власника файлу
   - `chgrp` - зміна групи файлу

6. **Інформація:**
   - `stat` - детальна інформація про файл
   - `df` - інформація про використання диска
   - `du` - розмір директорій
   - `file` - визначення типу файлу

### Питання 2: Як створити нового користувача в Linux?

**Відповідь:**

Для створення нового користувача використовується команда `useradd`:

```bash
sudo useradd -m -s /bin/bash username
```

**Параметри:**
- `-m` - створює домашню директорію користувача
- `-s /bin/bash` - встановлює оболонку за замовчуванням
- `username` - ім'я нового користувача

**Додаткові параметри:**
- `-c "Full Name"` - повне ім'я користувача
- `-d /home/custom` - кастомна домашня директорія
- `-g groupname` - основна група
- `-G group1,group2` - додаткові групи
- `-u UID` - встановлення конкретного UID

**Встановлення пароля:**
```bash
sudo passwd username
```

**Модифікація користувача:**
```bash
sudo usermod -aG sudo username  # Додати до групи sudo
sudo usermod -s /bin/zsh username  # Змінити оболонку
```

**Видалення користувача:**
```bash
sudo userdel username  # Без домашньої директорії
sudo userdel -r username  # З видаленням домашньої директорії
```

### Питання 3: Які команди використовуються для роботи з мережею?

**Відповідь:**

Основні команди для роботи з мережею:

1. **Налаштування інтерфейсів:**
   - `ip` - сучасна команда для налаштування мережі
     - `ip addr show` - показ адрес
     - `ip link show` - показ інтерфейсів
     - `ip route show` - показ маршрутів
   - `ifconfig` - застаріла команда (deprecated)

2. **Перевірка з'єднання:**
   - `ping` - перевірка доступності хоста
   - `traceroute` / `tracepath` - трасування маршруту
   - `mtr` - комбінація ping та traceroute

3. **DNS:**
   - `nslookup` - DNS запити
   - `dig` - детальні DNS запити
   - `host` - перетворення імені в IP

4. **Мережеві з'єднання:**
   - `ss` - показ мережевих з'єднань (сучасна заміна netstat)
   - `netstat` - показ мережевих з'єднань (застаріла)
   - `lsof -i` - відкриті мережеві файли

5. **Мережеві утиліти:**
   - `curl` - HTTP клієнт
   - `wget` - завантаження файлів
   - `nc` / `netcat` - мережевий інструмент
   - `tcpdump` - аналіз мережевого трафіку
   - `wireshark` - графічний аналіз трафіку

6. **Файрвол:**
   - `iptables` - налаштування файрволу
   - `ufw` - спрощений файрвол (Ubuntu)
   - `firewalld` - файрвол для systemd

### Питання 4: Як перевірити статус сервісу в systemd?

**Відповідь:**

**Перевірка статусу конкретного сервісу:**
```bash
systemctl status servicename
```

**Основні команди управління сервісами:**

1. **Перевірка статусу:**
   ```bash
   systemctl status sshd
   systemctl is-active sshd
   systemctl is-enabled sshd
   ```

2. **Управління сервісом:**
   ```bash
   sudo systemctl start servicename    # Запуск
   sudo systemctl stop servicename     # Зупинка
   sudo systemctl restart servicename  # Перезапуск
   sudo systemctl reload servicename   # Перезавантаження конфігурації
   ```

3. **Автозапуск:**
   ```bash
   sudo systemctl enable servicename   # Увімкнути автозапуск
   sudo systemctl disable servicename  # Вимкнути автозапуск
   ```

4. **Перегляд сервісів:**
   ```bash
   systemctl list-units --type=service              # Всі сервіси
   systemctl list-units --type=service --state=running  # Активні
   systemctl list-unit-files --type=service         # Всі файли сервісів
   systemctl list-unit-files --type=service --state=enabled  # Автозапуск
   ```

5. **Логи:**
   ```bash
   journalctl -u servicename           # Логи сервісу
   journalctl -u servicename -f        # Слідкування за логами
   journalctl -u servicename --since today  # Логи за сьогодні
   ```

### Питання 5: Як створити bash-скрипт та надати йому права на виконання?

**Відповідь:**

**Крок 1: Створення файлу скрипта**
```bash
nano script.sh
# або
vim script.sh
```

**Крок 2: Додати shebang на початку файлу**
```bash
#!/bin/bash
```
Це вказує системі, який інтерпретатор використовувати.

**Крок 3: Додати код скрипта**
```bash
#!/bin/bash
echo "Привіт, світ!"
date
whoami
```

**Крок 4: Надати права на виконання**
```bash
chmod +x script.sh
```

Альтернативно можна встановити конкретні права:
```bash
chmod 755 script.sh  # rwxr-xr-x
chmod u+x script.sh  # Тільки для власника
```

**Крок 5: Виконання скрипта**

Варіант 1 (якщо права встановлено):
```bash
./script.sh
```

Варіант 2 (без прав на виконання):
```bash
bash script.sh
```

Варіант 3 (з повним шляхом):
```bash
/повний/шлях/до/script.sh
```

**Додаткові поради:**

1. **Перевірка синтаксису:**
   ```bash
   bash -n script.sh
   ```

2. **Відлагодження:**
   ```bash
   bash -x script.sh  # Показує кожну виконувану команду
   ```

3. **Структура скрипта:**
   ```bash
   #!/bin/bash
   # Коментар з описом скрипта
   
   # Змінні
   VAR="значення"
   
   # Функції
   function my_function() {
       echo "Функція виконана"
   }
   
   # Основна логіка
   echo "Початок виконання"
   my_function
   echo "Кінець виконання"
   ```

4. **Обробка помилок:**
   ```bash
   # Зупинка при помилці
   set -e
   
   # Або з детальним виводом
   set -xe
   ```

### Питання 7: Як знайти файли за різними критеріями?

**Відповідь:**

```bash
# За ім'ям
find /path -name "filename"

# За розширенням
find /path -name "*.txt"

find /path -user username

find /path -name "*.log" -size +10M -mtime -7
```
